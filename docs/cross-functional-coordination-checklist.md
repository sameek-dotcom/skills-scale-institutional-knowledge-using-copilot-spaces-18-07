# Cross-Functional Coordination Checklist

This checklist supports effective coordination and collaboration across multiple teams and departments working on shared projects and work streams.

---

## Pre-Project Coordination Setup

### Dependency Mapping
- [ ] Identify all teams and departments involved in the project
- [ ] Map interdependencies between teams and work streams
- [ ] Document critical path dependencies and sequencing requirements
- [ ] Create dependency matrix showing which teams depend on which deliverables
- [ ] Identify potential bottlenecks and single points of failure
- [ ] Establish dependency tracking mechanism (shared tool/board)
- [ ] Schedule initial cross-team kickoff meeting

### Communication Framework
- [ ] Define coordination meeting cadence (daily, weekly, bi-weekly)
- [ ] Establish escalation contacts from each team
- [ ] Create shared communication channels (Slack, Teams, etc.)
- [ ] Document communication protocols and response time expectations
- [ ] Set up status reporting templates and schedules
- [ ] Define decision-making authority and approval processes
- [ ] Establish handoff procedures and sign-off requirements

### Resource Planning
- [ ] Document shared resources and their allocation
- [ ] Create resource availability matrix
- [ ] Establish priority framework for resource conflicts
- [ ] Define escalation process for resource allocation disputes
- [ ] Create contingency plan for resource unavailability
- [ ] Schedule resource capacity review meetings

---

## Ongoing Coordination Activities

### Regular Meetings
- [ ] Conduct weekly cross-functional sync meetings
- [ ] Maintain and update dependency tracking during each meeting
- [ ] Review blockers and escalation items
- [ ] Confirm handoff readiness and acceptance criteria
- [ ] Discuss upcoming dependencies and sequencing
- [ ] Document decisions and action items
- [ ] Publish meeting notes to all stakeholders

### Dependency Management
- [ ] Review dependency status at start of each sprint/cycle
- [ ] Flag any dependencies at risk or delayed
- [ ] Coordinate mitigation plans for at-risk dependencies
- [ ] Communicate dependency status changes to all affected teams
- [ ] Escalate critical dependency issues to leadership
- [ ] Update dependency tracking system in real-time
- [ ] Document lessons learned on dependency management

### Quality Handoffs
- [ ] Verify handoff acceptance criteria before transfer
- [ ] Conduct handoff quality review meeting
- [ ] Document deliverable specifications and assumptions
- [ ] Provide knowledge transfer/training on handed-off work
- [ ] Confirm receiving team understanding and readiness
- [ ] Establish support contact for post-handoff questions
- [ ] Schedule post-handoff check-in meeting

### Issue Resolution
- [ ] Establish process for raising cross-team issues
- [ ] Create shared issue/blocker tracking system
- [ ] Conduct issue triage meetings when blockers emerge
- [ ] Assign ownership for issue resolution with clear timelines
- [ ] Track resolution progress and communicate updates
- [ ] Escalate unresolved issues to department leads
- [ ] Document issue resolution patterns for process improvement

---

## Phase-Specific Coordination Checkpoints

### Initiation Phase
- [ ] All teams representatives present at project kickoff
- [ ] Shared understanding of project goals and success criteria
- [ ] Cross-team dependencies clearly identified and documented
- [ ] Coordination roles and responsibilities assigned
- [ ] Team leads confirm resource availability and commitments
- [ ] Communication and escalation framework established
- [ ] Initial risk assessment includes cross-team risks

### Planning Phase
- [ ] Detailed work stream plans account for dependencies
- [ ] Cross-team sequencing and milestones aligned
- [ ] Resource conflicts identified and resolved
- [ ] Contingency plans created for critical dependencies
- [ ] Integration points clearly defined
- [ ] Handoff schedules confirmed with receiving teams
- [ ] Test and validation plans include cross-team scenarios

### Execution Phase
- [ ] Weekly dependency status reviews conducted
- [ ] Blockers identified and escalated promptly
- [ ] Cross-team coordination metrics tracked (blocker resolution time, handoff quality)
- [ ] Mid-project risk assessment completed
- [ ] Lessons learned on coordination captured and shared
- [ ] Resource allocation conflicts resolved
- [ ] Communication cadence maintained

### Testing & Deployment Phase
- [ ] Integration testing plan includes all cross-team deliverables
- [ ] Deployment sequence coordinated across teams
- [ ] Rollback procedures address cross-team impacts
- [ ] Support team ready with cross-team escalation contacts
- [ ] Post-deployment validation involves relevant teams
- [ ] Communication plan includes cross-team status updates
- [ ] Incident response plan covers cross-team coordination

### Retrospective Phase
- [ ] Cross-functional retrospective conducted
- [ ] Coordination effectiveness assessed against metrics
- [ ] Dependency management lessons documented
- [ ] Communication effectiveness reviewed
- [ ] Process improvements identified for future projects
- [ ] Best practices on cross-team collaboration captured
- [ ] Action items assigned for process improvements

---

## Cross-Functional Coordinator Responsibilities

### Preparation
- [ ] Coordinator identified and assigned at project start
- [ ] Coordinator understands all teams' roles and constraints
- [ ] Coordinator has access to all communication channels and tracking systems
- [ ] Coordinator briefed on project goals, timeline, and dependencies
- [ ] Coordinator establishes relationships with team leads
- [ ] Coordinator creates coordination communication plan

### Coordination Activities
- [ ] Maintain current dependency status at all times
- [ ] Facilitate resolution of cross-team conflicts
- [ ] Ensure escalations are addressed promptly
- [ ] Provide regular status updates to Project Manager
- [ ] Flag emerging risks or bottlenecks proactively
- [ ] Coordinate schedule adjustments when dependencies slip
- [ ] Support team leads in resolving resource conflicts
- [ ] Document coordination decisions and rationale

### Reporting & Communication
- [ ] Prepare weekly cross-functional status report
- [ ] Highlight key dependencies coming due
- [ ] Report blocker resolution rates and trends
- [ ] Communicate resource allocation and conflicts
- [ ] Share lessons learned and best practices
- [ ] Escalate critical issues to Project Manager and leadership
- [ ] Maintain accessible shared documentation

---

## Success Metrics

### Tracking Coordination Effectiveness
- [ ] Percentage of dependencies met on schedule
- [ ] Average blocker resolution time
- [ ] Number of escalations and their resolution time
- [ ] Handoff quality score (meeting acceptance criteria)
- [ ] Team satisfaction with coordination (survey/feedback)
- [ ] Number of unplanned rework items due to coordination gaps
- [ ] Schedule variance due to cross-team dependencies
- [ ] Communication effectiveness rating

### Review Cadence
- [ ] Weekly review of dependency status
- [ ] Bi-weekly assessment of blocker metrics
- [ ] Monthly cross-functional effectiveness review
- [ ] Project-end retrospective on coordination
- [ ] Post-project lessons learned documentation

## Tools & Resources

### Recommended Coordination Tools
- Dependency tracking spreadsheet or Jira portfolio view
- Shared project board or timeline (Asana, Monday.com, etc.)
- Communication platform (Slack, Teams, Discord)
- Meeting scheduling and notes (Confluence, Notion)
- Status dashboard or automated reporting
- Issue/blocker tracking system

### Documentation Templates
- Dependency matrix template
- Cross-team risk register
- Handoff acceptance criteria template
- Coordination meeting notes template
- Escalation tracking template
- Lessons learned template
