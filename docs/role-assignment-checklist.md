# Role Assignment and Accountability Checklist

Use this checklist when establishing roles and responsibilities for new projects. Ensure all critical roles are assigned and stakeholders understand their responsibilities.

---

## Project Initiation Phase

### Core Roles Assignment
- [ ] **Project Manager**: Assigned and available for project kickoff
- [ ] **Product Manager**: Defined product vision and success metrics
- [ ] **Development Lead**: Ready to provide technical estimates and planning input

### Supporting Roles Assignment (based on project scope)
- [ ] **Stakeholder Liaison**: Identified if multiple stakeholder groups or complex communication needs
- [ ] **Quality Assurance Owner**: Assigned if testing strategy required
- [ ] **Compliance & Risk Officer**: Assigned if regulatory/compliance risks identified
- [ ] **Cross-Functional Coordinator**: Assigned if project spans multiple teams/departments
- [ ] **Knowledge Management Owner**: Assigned for projects with significant learnings or process impact

### Role Clarification
- [ ] All assigned roles have documented responsibilities (reference: octoacme-roles-and-personas.md)
- [ ] Escalation paths defined for each role
- [ ] Key interactions between roles understood by all stakeholders
- [ ] Coverage determined for 24x7 support requirements (if applicable)

---

## Planning Phase

### Communication Plan Review
- [ ] Stakeholder Liaison has defined communication cadence with stakeholders
- [ ] Project Manager has established team communication protocols
- [ ] Cross-Functional Coordinator has identified dependencies and communication touchpoints

### Quality and Compliance Planning
- [ ] QA Owner has developed testing strategy and test plan
- [ ] Compliance & Risk Officer has completed risk assessment and compliance review
- [ ] Risk mitigation plans documented and assigned to responsible parties

### Knowledge Management Setup
- [ ] Knowledge Management Owner has defined documentation standards
- [ ] Project repository and knowledge base locations established
- [ ] Retrospective schedule and format defined

---

## Execution Phase

### Role Performance Monitoring
- [ ] Project Manager tracking team velocity and delivery progress
- [ ] QA Owner monitoring test coverage and defect trends
- [ ] Stakeholder Liaison managing stakeholder feedback and satisfaction
- [ ] Cross-Functional Coordinator tracking inter-team dependencies and blockers
- [ ] Compliance & Risk Officer monitoring risk register and compliance status

### Escalation Effectiveness
- [ ] Escalation paths being followed appropriately
- [ ] Issues resolved at proper management levels
- [ ] Critical risks and compliance issues flagged promptly

---

## Closeout Phase

### Knowledge Transfer and Documentation
- [ ] Knowledge Management Owner has scheduled retrospectives
- [ ] Lessons learned documented and accessible
- [ ] Project artifacts archived appropriately
- [ ] Process improvements identified and logged
- [ ] Team playbooks and runbooks updated

### Role Transition
- [ ] All responsibilities transferred to ongoing support team
- [ ] Documentation complete and handoff reviewed
- [ ] Stakeholder relationships transitioned (if needed)
- [ ] Knowledge base updated with project outcomes

---

## Role Interaction Verification

Verify these key interactions are functioning well:

### Project Manager interactions
- [ ] Regular sync with Stakeholder Liaison on stakeholder status
- [ ] Weekly quality updates from QA Owner
- [ ] Risk register reviews with Compliance & Risk Officer
- [ ] Dependency coordination with Cross-Functional Coordinator
- [ ] Lessons learned reviews with Knowledge Management Owner

### Product Manager interactions
- [ ] Alignment with Stakeholder Liaison on customer feedback
- [ ] Review of Quality standards with QA Owner
- [ ] Collaboration with Cross-Functional Coordinator on feature dependencies
- [ ] Input to retrospectives from Knowledge Management Owner

### Development Team interactions
- [ ] Clear technical requirements from Product Manager
- [ ] Quality expectations from QA Owner
- [ ] Compliance requirements from Compliance & Risk Officer
- [ ] Coordination touchpoints from Cross-Functional Coordinator

---

## Notes

Use this checklist for:
- New project launches
- Mid-project staffing changes
- Organizational restructuring
- Post-project reviews to validate role effectiveness
